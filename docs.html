Integrasi Layanan OTP
ke Aplikasi Kamu

API ini memungkinkan kamu untuk mengintegrasikan layanan virtual number OTP Kayz Market ke website, bot, atau aplikasi kamu. Beli nomor OTP, cek kode, kelola saldo — semua lewat REST API.

Base URL
Autentikasi

Semua endpoint yang bersifat privat memerlukan API key yang dikirim melalui header x-api-key. API key didapatkan setelah registrasi dan bisa dilihat di endpoint /api/user/apikey atau di profil akun.

Header: x-api-key: DOTP-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

Endpoint tanpa label 🔒 Auth bisa diakses tanpa token. Request yang memerlukan autentikasi akan mengembalikan 401 jika token tidak valid.

Server OTP

Kayz Market menyediakan 2 server OTP yang dapat kamu pilih saat melakukan pemesanan nomor.

Server 1
Kayz Market
server: "server1"

Server dengan stok besar dan harga kompetitif. Mendukung banyak negara dan platform.

Server 2
Kayz Market
server: "server2"

Spesialisasi nomor Indonesia dengan pilihan operator (Telkomsel, Indosat, dll).

POST /api/user/register Daftar akun baru

Mendaftarkan akun baru untuk mendapatkan akses ke API. Setelah registrasi, gunakan endpoint login untuk mendapatkan token.

Body (application/json)
FieldTipeWajibKeterangan
usernamestringYa3–20 karakter, huruf kecil, angka, underscore
passwordstringYaMinimal 6 karakter
emailstringYaAlamat email valid
whatsappstringYaNomor WhatsApp 8–16 digit
register.sh
curl -X POST BASE_URL_PLACEHOLDER/api/user/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "botku",
    "password": "password123",
    "email": "bot@example.com",
    "whatsapp": "08123456789"
  }'
register.js
const res = await fetch("BASE_URL_PLACEHOLDER/api/user/register", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    username: "botku",
    password: "password123",
    email: "bot@example.com",
    whatsapp: "08123456789"
  })
});
const data = await res.json();
// { ok: true, message: "Akun berhasil dibuat. Silakan login." }
GET /api/user/apikey Lihat API key 🔒 Auth

Mengembalikan API key aktif milik akun. API key sudah otomatis digenerate saat register. Gunakan nilai apiKey ini di header x-api-key untuk semua request OTP.

Jangan bagikan API key ke siapapun. Jika bocor, segera regenerate.

response.json
{
  "ok": true,
  "apiKey": "DOTP-A1B2C3D4E5F6G7H8I9J0K1L2M3N4O5P6",
  "generatedAt": 1716000000000
}
POST /api/user/apikey/regenerate Generate ulang API key 🔒 Auth

Generate API key baru. Key lama akan langsung tidak aktif — semua integrasi yang memakai key lama harus diperbarui.

response.json
{
  "ok": true,
  "apiKey": "DOTP-Z9Y8X7W6V5U4T3S2R1Q0P9O8N7M6L5K4",
  "message": "API key berhasil di-generate ulang. Perbarui key di semua integrasi kamu."
}

Pesanan OTP
1
Ambil katalog produk
Gunakan endpoint negara & layanan untuk mendapatkan product_id (S1) atau number_id + provider_id (S2)
2
Beli nomor
Kirim request ke POST /api/otp/order dengan data dari langkah 1
3
Poll status OTP
Cek GET /api/otp/order/:id/status setiap 5–10 detik sampai status completed
4
Finish atau Cancel
Setelah OTP diterima, kirim finish. Jika tidak terima OTP sebelum expired, kirim cancel untuk refund
POST /api/otp/order Beli nomor OTP 🔒 Auth

Membeli nomor virtual OTP. Saldo akan dipotong sesuai harga produk. Gunakan parameter sesuai server yang dipilih.

Body — Server 1
FieldTipeWajibKeterangan
providerstringYaNilai: "server1"
product_idintegerYaID produk dari /api/otp/server1/products
Body — Server 2
FieldTipeWajibKeterangan
providerstringYaNilai: "server2"
number_idintegerYanumber_id dari /api/otp/countries/:serviceId
provider_idstringYaprovider_id dari pricelist
operator_idintegerTidakID operator (default: any)
response.json
{
  "ok": true,
  "orderId": "OTP-1716000000-a1b2c3d4",
  "phoneNumber": "+6281234567890",
  "expiresAt": 1716001200000,
  "balance": 46500
}

Saldo dipotong langsung saat order berhasil. Jika OTP tidak diterima sebelum expired_at, kirim cancel untuk refund.

GET /api/otp/order/:id/status Cek OTP & status 🔒 Auth

Cek status pesanan dan dapatkan kode OTP jika sudah diterima. Poll endpoint ini setiap 5–10 detik.

Status yang mungkin
StatusKeterangan
waitingMenunggu OTP masuk
completedOTP sudah diterima — cek field otp
canceledDibatalkan, saldo sudah dikembalikan
expiredWaktu habis tanpa OTP, saldo dikembalikan
cancelingPembatalan sedang diproses
response.json (OTP diterima)
{
  "ok": true,
  "status": "completed",
  "otp": "949708",
  "otpMsg": "Your WhatsApp code: 949-708\nDon't share this code",
  "phoneNumber": "+6281234567890"
}
// Masih menunggu OTP:
{
  "ok": true,
  "status": "waiting",
  "otp": null,
  "phoneNumber": "+6281234567890"
}
POST /api/otp/order/:id/cancel Cancel pesanan 🔒 Auth

Membatalkan pesanan OTP yang masih aktif. Saldo akan dikembalikan. Hanya bisa cancel pesanan yang statusnya ACTIVE.

response.json
{ "ok": true, "message": "Pesanan berhasil dibatalkan." }
POST /api/otp/order/:id/finish Selesaikan pesanan 🔒 Auth

Menandai pesanan sebagai selesai setelah OTP berhasil digunakan. Ini memberitahu sistem bahwa nomor sudah tidak digunakan lagi.

response.json
{ "ok": true, "message": "Pesanan selesai." }
POST /api/otp/order/:id/resend Resend SMS 🔒 Auth

Meminta pengiriman ulang SMS OTP ke nomor yang sama. Hanya tersedia di Server 1.

response.json
{ "ok": true, "message": "SMS berhasil dikirim ulang." }

Deposit / Top Up Saldo

Tambah saldo akun melalui QRIS atau metode pembayaran lain yang tersedia.

POST /api/deposit/create Buat transaksi deposit 🔒 Auth

Membuat transaksi deposit baru. Mengembalikan QR code QRIS yang bisa ditampilkan ke pengguna untuk dibayar.

Body (application/json)
FieldTipeWajibKeterangan
amountintegerYaJumlah saldo yang ingin ditambahkan (dalam Rupiah)
response.json
{
  "ok": true,
  "depId": "DEP-1716000000-abc1",
  "amount": 50000,
  "adminFeeDeposit": 500,
  "totalBayarDeposit": 50500,
  "qr": "BASE_URL_PLACEHOLDER/image/qr/DEP-...",
  "qrString": "00020101021226670016...",
  "expiredAt": 1716003600000

Tampilkan qr sebagai gambar atau gunakan qrString untuk generate QR sendiri. Poll status deposit menggunakan depId.
